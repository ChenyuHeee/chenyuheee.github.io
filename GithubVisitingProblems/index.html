    针对访问Github出现问题的MacOS解决方案

    在 Mac 上访问 GitHub 时断时续确实很让人头疼。这通常和网络连接、DNS解析、系统或浏览器设置有关。别担心，我们可以按照下表的步骤来排查和解决。

    可能原因 排查思路 解决方法
    DNS 解析问题 🎯 最常见的原因之一，导致域名无法正确转换为IP地址。 1. 清除DNS缓存。
    2. 更换公共DNS。
    3. 修改Hosts文件。
    网络连接与代理设置 🌐 网络本身不稳定，或代理设置干扰了正常连接。 1. 检查网络连接。
    2. 排查代理设置。
    3. 调整防火墙。
    浏览器与系统缓存 🖥️ 浏览器或系统的陈旧缓存可能导致页面加载异常。 1. 清除浏览器缓存。
    2. 尝试其他浏览器。
    GitHub 服务问题 🤔 偶尔也可能是 GitHub 服务器本身出现故障或维护。 访问 GitHub Status 官方状态页面进行确认。

    🛠️ 详细操作指南¶

    DNS 解析问题¶

    这是导致无法访问的最常见原因，可以尝试以下方法：

    1. 清除DNS缓存 打开"终端"应用程序，输入以下命令后回车，并输入你的电脑密码（输入时不可见）： 

      sudo dscacheutil -flushcache
      对于 macOS 系统，此命令可以清除已有的DNS缓存。

    2. 更换公共DNS服务器 这可以解决因默认DNS服务器解析不佳或污染导致的问题。

      • 进入"系统偏好设置" -> "网络"。
      • 选择你正在使用的网络连接（如Wi-Fi），点击"高级"按钮。
      • 切换到"DNS"选项卡，在左侧DNS服务器列表中，点击下方的"+"号，添加以下可靠的DNS地址（建议同时添加IPv4和IPv6）：
        • 8.8.8.8 和 8.8.4.4 （Google Public DNS）
        • 1.1.1.1 和 1.0.0.1 （Cloudflare DNS）
        • 223.5.5.5 和 223.6.6.6 （阿里云DNS）
      • 添加后，点击"好" -> "应用"。

    3. 修改Hosts文件（终极方法） 此方法通过直接绑定域名与正确IP地址，能有效绕过DNS污染。

      • 获取最新IP：使用浏览器访问 IPAddress.com 网站（例如 https://github.com.ipaddress.com/ 或 https://www.ipaddress.com/）查询以下域名的当前真实IP：
        • github.com
        • github.global.ssl.fastly.net
        • assets-cdn.github.com
        • codeload.github.com
        • raw.githubusercontent.com
      • 修改Hosts文件：
        • 打开"终端"，输入 sudo nano /etc/hosts 并回车。
        • 输入密码后，会用nano编辑器打开hosts文件。
        • 在文件末尾，按以下格式添加查询到的IP和域名（IP地址请以你实际查询到的为准）： 
          140.82.113.4 github.com
199.232.69.194 github.global.ssl.fastly.net
185.199.108.153 assets-cdn.github.com
140.82.114.9 codeload.github.com
185.199.108.133 raw.githubusercontent.com
        • 编辑完成后，按 Control + O 回车保存，再按 Control + X 退出编辑器。
      • 刷新DNS缓存：在终端再次执行 sudo dscacheutil -flushcache 或 sudo killall -HUP mDNSResponder 使新的hosts文件生效。

    网络连接与代理设置¶

    1. 检查网络连接：确保你的网络连接正常，可以尝试访问其他网站（如 apple.com）来确认。如果条件允许，切换至其他网络（如手机热点）测试是否为当前网络问题。

    2. 排查代理设置：

      • 如果你没有使用代理，请检查：进入"系统偏好设置" -> "网络" -> 选择你的网络 -> "高级" -> "代理"，确保所有代理协议（如Web代理、安全Web代理等）都未被勾选。
      • 如果你正在使用代理（包括VPN或科学上网工具），请确保其配置正确且工作正常。有时关闭代理或更换节点可以解决问题。

    3. 调整防火墙：暂时禁用Mac的防火墙进行测试。进入"系统偏好设置" -> "安全性与隐私" -> "防火墙"，确保防火墙已关闭或GitHub相关应用不在阻止列表中。

    浏览器与系统缓存¶

    有时浏览器缓存的数据出错也会导致页面无法加载。

    • 清除浏览器缓存：根据你使用的浏览器（Safari, Chrome, Firefox等），在其设置中找到"清除浏览数据"或类似选项，选择清除缓存和Cookie。
    • 尝试其他浏览器：用另一个浏览器访问GitHub，如果正常，则问题很可能出在原浏览器的设置或扩展上。

    💡 备选方案¶

    如果上述方法尝试后问题依旧，或者你急需下载代码，可以考虑以下备选路径：

    • 使用GitHub镜像站点：国内有一些GitHub镜像站，例如 https://github.com.cnpmjs.org/ 或 https://hub.fastgit.org/。你可以直接访问这些镜像站，或者在使用Git克隆时，将仓库地址中的 github.com 替换为镜像站的域名。
    • 使用加速工具：有一些专门为开发者设计的开源工具，如 DevSidecar 或 Watt Toolkit，它们可以提供更稳定的访问加速服务。
